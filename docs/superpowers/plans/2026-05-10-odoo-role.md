# Rôle Odoo — Plan d'implémentation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Déployer Odoo 19 Community Edition sur erp01.adlin.lab avec PostgreSQL local (peer auth), workers multi-process, et le module auth_ldap activé pour l'authentification FreeIPA.

**Architecture:** Odoo 19 CE via RPM nightly officiel + python3.11 (EPEL). PostgreSQL local avec peer authentication (pas de mot de passe). Pas de TLS sur erp01 — proxy01 termine le TLS et forwarde en HTTP vers 8069 (UI) et 8072 (longpolling). Module auth_ldap activé via CLI Odoo, configuration LDAP manuelle post-déploiement dans l'UI.

**Tech Stack:** Rocky Linux 9, Odoo 19 CE (nightly RPM), PostgreSQL 13, python3.11 (EPEL), community.postgresql, ansible.posix.

**Files:**

- [ ] **Étape 4.1 : Créer roles/odoo/tasks/database.yml**

```yaml
---
# roles/odoo/tasks/database.yml
#
# Peer authentication : l'utilisateur système odoo (créé par le RPM) correspond
# au rôle PostgreSQL odoo. Connexion via socket Unix sans mot de passe.

- name: "database | Installer PostgreSQL"
  ansible.builtin.dnf:
    name: postgresql-server
    state: present

- name: "database | Initialiser le cluster PostgreSQL"
  ansible.builtin.command:
    cmd: postgresql-setup --initdb
  args:
    creates: /var/lib/pgsql/data/PG_VERSION

- name: "database | Démarrer et activer PostgreSQL"
  ansible.builtin.systemd:
    name: postgresql
    state: started
    enabled: true

- name: "database | Créer le rôle PostgreSQL odoo (peer auth — sans mot de passe)"
  community.postgresql.postgresql_user:
    name: "{{ odoo_db_user }}"
    role_attr_flags: LOGIN
    state: present
  become_user: postgres

- name: "database | Créer la base de données odoo"
  community.postgresql.postgresql_db:
    name: "{{ odoo_db_name }}"
    owner: "{{ odoo_db_user }}"
    encoding: UTF8
    state: present
  become_user: postgres
```

### Task 7 : tasks/init.yml

**Files:**
- Create: `roles/odoo/tasks/init.yml`

- [ ] **Étape 7.1 : Créer roles/odoo/tasks/init.yml**

```yaml
---
# roles/odoo/tasks/init.yml
#
# Initialise la base Odoo et active auth_ldap — chaque opération est protégée
# par un guard idempotent.
#
# --workers=0 force le mode single-thread pour l'initialisation, indépendamment
# de la valeur dans odoo.conf (qui serait active pour le service daemon).
#
# Séquence :
#   1. Vérifier si la base est initialisée (table ir_module_module)
#   2. Si non : --init base --stop-after-init
#   3. Vérifier si auth_ldap est installé
#   4. Si non : -i auth_ldap --stop-after-init
#   5. Démarrer et activer le service odoo

- name: "init | Vérifier si la base Odoo est initialisée (table ir_module_module)"
  ansible.builtin.command:
    cmd: >
      psql -d {{ odoo_db_name }} -tAc
      "SELECT EXISTS (SELECT 1 FROM information_schema.tables
       WHERE table_schema='public' AND table_name='ir_module_module')"
  become_user: odoo
  register: odoo_db_initialized
  changed_when: false
  failed_when: false

- name: "init | Initialiser la base Odoo (--init base)"
  ansible.builtin.command:
    cmd: >
      odoo -d {{ odoo_db_name }}
      --init base
      --workers=0
      --stop-after-init
      --config {{ odoo_config_file }}
  become_user: odoo
  when: "odoo_db_initialized.stdout.strip() != 't'"
  changed_when: true

- name: "init | Vérifier si le module auth_ldap est installé"
  ansible.builtin.command:
    cmd: >
      psql -d {{ odoo_db_name }} -tAc
      "SELECT state FROM ir_module_module WHERE name='auth_ldap'"
  become_user: odoo
  register: odoo_auth_ldap_state
  changed_when: false

- name: "init | Activer le module auth_ldap"
  ansible.builtin.command:
    cmd: >
      odoo -d {{ odoo_db_name }}
      -i auth_ldap
      --workers=0
      --stop-after-init
      --config {{ odoo_config_file }}
  become_user: odoo
  when: "odoo_auth_ldap_state.stdout.strip() != 'installed'"
  changed_when: true

- name: "init | Démarrer et activer le service Odoo"
  ansible.builtin.systemd:
    name: odoo
    state: started
    enabled: true

- name: "init | Afficher les instructions de configuration LDAP post-déploiement"
  ansible.builtin.debug:
    msg:
      - "Odoo déployé sur https://{{ odoo_fqdn_public }}"
      - "Configurer LDAP manuellement : Settings → Technical → LDAP Servers"
      - "  LDAP Server     : ldaps://ipa01.adlin.lab"
      - "  Port            : 636"
      - "  Bind DN         : {{ odoo_ldap_bind_dn }}"
      - "  Base DN         : {{ odoo_ldap_base }}"
      - "  Filter          : {{ odoo_ldap_filter }}"
      - "  TLS             : cocher 'Use TLS'"
```
